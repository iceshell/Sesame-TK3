   - 便于追踪和回顾

   - Day 1-2 → Day 3-4 → Day 5-7
   - 每个阶段有明确交付物
   - 风险可控

3. **质量优先**
   - 代码质量检查在构建之前
   - 避免带问题发布
   - 长期收益大

4. **Git规范**

   - 部分检查可以自动化
